# Scientific-review reports

A ten-phase review that moves this project from a well-packaged manuscript toward a
scientifically defensible artifact. Each report explains *why* a change was needed,
*which* scientific risk it addresses, *which* files changed, and *how* it improves
publication readiness. Read in order:

| Phase | Report | What it does |
|------:|--------|--------------|
| 1 | [AUDIT_REPORT.md](AUDIT_REPORT.md) | Repository/manuscript/pipeline maps; strengths, weaknesses, publication blockers; reproducibility verified. |
| 2 | [CLAIM_AUDIT.md](CLAIM_AUDIT.md) | Every claim classified A/B/C/D with evidence and revisions. |
| 3 | [RQ4_FIX_REPORT.md](RQ4_FIX_REPORT.md) | Removes the "mitigation preserves item quality" overclaim. |
| 4 | [STATISTICAL_VALIDITY_REPORT.md](STATISTICAL_VALIDITY_REPORT.md) | Fixes option-independence; conditional logit + cluster-robust SEs + cluster bootstrap. |
| 5 | [CLOSED_LOOP_ANALYSIS.md](CLOSED_LOOP_ANALYSIS.md) | Removes wording implying real-world validation; names the synthetic circularity. |
| 6 | [REAL_COLLECTION_PLAN.md](REAL_COLLECTION_PLAN.md) | Modular real-collection package + operating protocol. |
| 7 | [EXPLOITABILITY_ANALYSIS.md](EXPLOITABILITY_ANALYSIS.md) | Predictive-validation pipeline + preregistration (results withheld). |
| 8 | [CITATION_AUDIT.md](CITATION_AUDIT.md) | All references verified real; incomplete metadata completed. |
| 9 | [TITLE_RECOMMENDATIONS.md](TITLE_RECOMMENDATIONS.md) | Five title framings; the unsupported "threat" assertion removed. |
| 10 | [VENUE_STRATEGY.md](VENUE_STRATEGY.md) | Acceptance odds, requirements, risks, objections per venue type. |

**One-line takeaway:** the artifact is honest and reproducible but *methodological*,
not empirical. The two removable publication blockers (invalid statistics,
claim–evidence gaps) are fixed here; the remaining *evidentiary* blockers are
bounded honestly and equipped with the infrastructure + preregistered protocols
needed to earn the empirical claims later.
